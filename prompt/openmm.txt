=== Monte Carlo Glass Simulation Context (OpenMM + Numba Hybrid GPU) ===
PROJECT

Hybrid GPU Monte Carlo Simulation of SiO₂–CaO–P₂O₅ Glass

GOAL

Implement a high‑performance NPT Monte Carlo simulation using:

• OpenMM GPU engine for energy calculations

• Numba‑accelerated Monte Carlo kernels

• Python orchestration for simulation control

This hybrid architecture enables GPU‑accelerated energy evaluation

while preserving flexible Monte Carlo move logic.

INPUT STRUCTURE

File: Ternary.txt

Format:

Line 1

Number of atoms

Line 2

Metadata

Line 3+

Element x y z

Example:

Si 5.238 10.221 7.331

O 6.112 11.003 8.114

The file contains a single atomic snapshot.

The implementation must include a robust parser that converts the

structure into arrays suitable for both:

• OpenMM System construction

• Numba Monte Carlo kernels

SYSTEM

Glass composition (molar %)

SiO₂ = 60%

CaO = 36%

P₂O₅ = 4%

Target density

2.2 g/cm³

Simulation box

30 Å × 30 Å × 30 Å

Atoms

~2000 atoms

Periodic boundary conditions enabled.

FORCE FIELD

Use Buckingham + Coulomb (Wolf summation).

Buckingham potential

V® = A exp(-r / ρ) − C / r⁶

Pairs

Si–O

Ca–O

P–O

O–O

Parameters loaded from external force‑field table.

COULOMB INTERACTION

Wolf electrostatics:

V_coul® = (q_i q_j / r) * erfc(alpha r)

Parameters:

cutoff = 10 Å

alpha = 0.2

Implementation:

Wolf Coulomb must be implemented inside an

OpenMM CustomNonbondedForce so it executes on the GPU.

MONTE CARLO ENSEMBLE

NPT Monte Carlo simulation.

Moves:

Atomic displacement move
Isotropic volume move
Acceptance criterion:

Metropolis algorithm.

Energy differences are evaluated by:

• updating OpenMM positions

• requesting potential energy from GPU

This allows GPU‑accelerated energy evaluation.

HYBRID GPU ARCHITECTURE

Energy evaluation → OpenMM GPU

Monte Carlo move logic → Numba kernels

Simulation controller → Python

Advantages:

• GPU energy evaluation

• flexible Monte Carlo moves

• minimal CPU overhead

Workflow per MC step:

Propose move (Numba)
Update coordinates
Push coordinates to OpenMM context
Request potential energy (GPU)
Apply Metropolis criterion
NEIGHBOR HANDLING

Neighbor lists are handled by OpenMM internal cutoff system.

PERFORMANCE STRATEGY

Use GPU for:

• pair interactions

• Coulomb calculations

• cutoff neighbor handling

Use Numba for:

• Monte Carlo move proposal

• coordinate perturbations

• RDF histogram kernels
